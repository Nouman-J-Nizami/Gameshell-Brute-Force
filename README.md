# SSH Brute Force Simulation & Linux Shell Lab

## Overview

This project combines two practical Linux and cybersecurity exercises:

1. a simulated brute force attack against SSH in a controlled Metasploitable lab;
2. a GameShell-based terminal practice session focused on Linux command-line fundamentals.

The goal was to understand authentication weaknesses, SSH hardening concepts, and core shell operations that are essential in cybersecurity and system administration.

---

## Tools Used

- Kali Linux
- Metasploitable
- SSH
- Nmap
- Python
- GameShell
- Bash
- Linux command-line utilities

---

## Part 1 – SSH Brute Force Simulation

### Activities Performed

- Verified SSH service availability on the target
- Tested network connectivity and port 22 exposure
- Troubleshot legacy SSH algorithm compatibility
- Built a Python script to simulate brute force logic using a wordlist
- Measured attempts and execution time
- Validated the risk of weak passwords in legacy environments

### Key Findings

- Weak credentials significantly reduce authentication security
- Legacy SSH configurations can create compatibility and security issues
- Rate limiting and key-based authentication are important defensive controls

---

## Part 2 – GameShell Linux Command-Line Lab

### Activities Performed

- Installed and launched GameShell on Kali Linux
- Completed 10 terminal missions
- Practiced filesystem navigation with `cd`
- Worked with `mkdir`, `mv`, `rm`, and `cp`
- Used wildcards and path handling
- Applied `ls -a` and tab completion
- Reinforced relative and absolute path usage

### Key Findings

- Strong command-line skills are essential for Linux administration
- Wildcards, autocompletion, and path handling improve efficiency
- Basic filesystem operations are foundational for cybersecurity workflows
